# Project Launch Planner

## Live Demo

[View the live project](https://mrmcgrain.github.io/NextChapter/)

## Problem

Students building a first software project often have the assignment requirements, but not a clear path for turning those requirements into a focused application, complete documentation, and a submission-ready checklist.

## Value

Project Launch Planner helps a student make stronger decisions before writing or submitting code. It guides them through the problem, user, value, smallest useful version, required features, future features, README draft, prompt history, and submission checklist.

## Project Plan

The smallest demonstration of value is a browser-based planner that saves a project idea, separates required and future features, generates a README draft, tracks meaningful AI prompts, and measures submission readiness.

The intended approach is to keep the first version simple: one static HTML, CSS, and JavaScript application that can run locally or on GitHub Pages without a build step.

## Features

### Completed

- Guided project planning form for problem, user, value, solution, and MVP.
- Required and future feature tracker.
- README draft generator based on the user's project plan.
- Prompt history tracker for AI collaboration evidence.
- Submission checklist with readiness score.
- Browser local storage for saved progress.
- Saved project switcher for managing multiple project plans in one browser.
- Responsive layout for desktop and mobile screens.

### Future Improvements

- Download README and prompt history as Markdown files.
- Export and import saved planner data.
- Add examples for different project types.
- Add a printable interview preparation view.

## Technologies Used

- HTML
- CSS
- JavaScript

## AI Tools Used

ChatGPT and Codex were used to interpret the project brief, choose a focused application idea, generate the initial implementation, refine the user experience, and prepare documentation. The project scope was kept small so the application demonstrates one complete user experience instead of many unfinished features.

## Running the Project

1. Clone or download the repository.
2. Open `index.html` in a web browser.
3. Fill out the planner and use the generated README draft, prompt history, and checklist while preparing the final submission.
